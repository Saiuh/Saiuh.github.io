Redis源码(4):字典

May 5, 2019 | redis

概述

字典又称为符号表或者关联数组、或映射（map），是一种用于保存键值对的抽象数据结构。字典中的每一个键 key 都是唯一的，通过 key 可以对值来进行查找或修改。C 语言中没有内置这种数据结构的实现，所以字典依然是 Redis自己构建的。

数据结构

redis中hash表是使用拉链法来解决哈希冲突，dictht代表一个哈希表，基本结构是一个dictEntry的数组，数组中的每一个元素（key, value）都是一个链表，哈希相同的数据在同一条链表上。

一个字典(dict)中有两个哈希表(dictht)，这是为了扩容的处理。

如果没有迭代器在运行则执行dictRehash()

是否需要扩容 _dictExpandIfNeeded

  1. 正在扩容则结束；
  2. 如果哈希表为空，则初始化为初始大小DICT_HT_INITIAL_SIZE（4）并结束；
  3. 哈希表满了，且(允许resize or 达到强制resize上限(已使用/容量>5))，则扩容并结束

获得key的位置(数组的index) _dictKeyIndex

  1. existing不为空，则将值设置为NULL;
  2. 调用_dictExpandIfNeeded(d)处理扩容；
  3. 遍历索引的哈希表（table[]）;
  4. 计算在哈希表中的index:hash值 & 掩码
  5. 如果这个位置已经存在这个key且existing不为空，则返回-1
  6. 其他情况返回index

添加key到字典中 dictAddRaw

  1. 如果正在rehash，则执行_dictRehashStep;
  2. 通过_dictKeyIndex获得存储位置，如果index返回-1表示已经存在（需要传existing参赛）则结束返回NULL;
  3. 是否在rehash，是则使用第二个哈希表(&d->ht[1]),否则使用第一个(&d->ht[0]);
  4. zmalloc分配内存空间;
  5. 将entry设置到table对应的index上，且处于冲突链表的头结点;
  6. 长度计数值加一;
  7. 将key设置进entry

添加key-value dictAdd

  1. 调用dictAddRaw函数添加key,value;
  2. 如果返回的entry为空，则返回DICT_ERR;
  3. 将value设置到entry中;
  4. 返回DICT_OK;

字典扩容 dictResize

  1. dict_can_resize不为0或字典正在rehash时，不扩容返回DICT_ERR;
  2. 获取当前已经使用的容量为minimal;
  3. 如果当前使用的容量小于初始容量，则minimal = 初始容量；
  4. 执行扩容dictExpand(d, minimal);

字典根据size扩容 dictExpand

  1. 正在rehash或当期使用的容量已经大于size了，则返回DICT_ERR;
  2. _dictNextPower计算根据当前size判断要扩容的大小，如果大于等于最大long值，则扩容到long_max+1; 如果小于初始大小，则扩容到初始大小；否则扩容到当前size的两倍;
  3. 当前字典的容量已经等于扩容值了，则返回DICT_ERR;
  4. 生成一个新哈希表n，长度为扩容值;
  5. 如果之前字典中哈希表0为空，则将n设置为哈希表0并返回DICT_OK;
  6. 否则将n设置为哈希表0;
  7. rehashidx字段设置为0;
  8. 返回DICT_OK;

重hash n个元素 dictRehash

n 代表本次哈希表中0中的n个元素rehash到哈希表1中。

  1. 初始容量值empty_visits = n*10;
  2. 如果字典不在rehash，则返回0;
  3. 遍历n–, 哈希表0使用量不等于0：
    1. 校验哈希表0的大小 > 字典的rehashidx字段;
    2. 循环：d->ht[0].table[d->rehashidx] == NULL，本质是找出table中不为null的那个元素，rehashidx记录这个元素的位置;
      1. rehashidx加一;
      2. empty_visits减一，如果empty_visits等于0，则返回1;
    3. 取出哈希表0中rehashidx位的元素;
    4. 循环：将取出的元素（链表）复制（逆向）到哈希表1中（位置通过hash计算），每复制一个链表节点，哈希表0.used减一，哈希表1.used加一;
    5. 哈希表0中rehashidx位元素置null;
    6. rehashidx加一;
  4. 校验是否rehash完成：如果used == 0，执行以下内容; 否则返回1
    1. 释放哈希表0的内存;
    2. 哈希表1执行初始设置(_dictReset);
    3. rehashidx 设置为-1;

限制时间的rehash dictRehashMilliseconds

一次rehash100个元素：

* 如果没超时就继续rehash
* 如果超时就返回本次rehash的元素个数
