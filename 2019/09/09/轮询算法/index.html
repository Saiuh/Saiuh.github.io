轮询方案

方案一

算法逻辑

  • 序号_index、一个当前权重current_weight;_
  • 循环
    • 计算新index=（index+1）/列表长度n
    • 列表中index位的元素的权重是否大于当前权重_current_weight，如果大于则选择该元素；_
    • _current_weight减去所有元素权重的最大公约数；_

_current_weight最开始设置为最大权重，如果current_weight小于等于0，则将其设置为最大权重。_
这个方式的特点就是找出第一个权重大于_current_weight的元素。_

特点

  • 计算最大公约数需要列表固定；
  • 权重大的在前就必须等权重大的分配完后才能分配后面的元素；

方案二

参考 Nginx 的平滑的加权轮询算法，平滑是相对一方案的轮询，方案一会发生权重大的一直分配的情况，平滑的加权轮询避免了这种情况。

算法逻辑

  1. 计算权重之和s，原始权重w[a, b, c]，元素权重w1[a, b, c]；
  2. 选出当前列表权重最大的元素，将该元素的权重减去s，得到新权重w2[a1, b, c];
  3. 将之前的原始w加上新的权重列表w2，得到w1[a+a1, 2b, 2c]
  4. 下次权重判断依旧从第2步开始；

示例：
元素[a, b, c] 权重[1, 2, 4]

选择前的权重w1 选中的元素 选择后的权重w2
[1, 2, 4] c [1, 2, -3]
[2, 4, 1] b [2, -3, 1]
[3, -1, 5] c [3, -1, -2]
[4, 1, 2] a [-3, 1, 2]
[-2, 3, 6] c [-2, 3, -1]
[-1, 5, 3] b [-1, -2, 3]
[0, 0, 7] c [0, 0, 0]

一个循环下来：a:b:c = 1:2:4，符合权重比例。

特点

  • 轮询平滑；
  • 需要轮询的列表固定；
  • 效率较高；

方案三

参考 dubbo RoundRobinLoadBalance 的实现方案：

dubbo是以请求的目标方法作为一个分组对实例（服务器）进行轮询；

健康分分配考虑以线索中的城市作为分组，对店铺进行轮询；

算法逻辑

  • 全局按照城市分组保存一个序号sequence，每次分配前递增；
  • 当前sequence对总权重取余mod；
  • 过滤店铺权重不一致：
    • 循环maxWeight次
      • 遍历店铺列表（mod和权重一起减，淘汰低优先级的店铺）
        • 如果mod为0，权重大于0，则选中当前店铺；
        • 当前店铺的权重减一，mod减一；
  • 过滤店铺权重一致：
    • 选中当前sequence对店铺列表的长度取余的序号对应的店铺；
